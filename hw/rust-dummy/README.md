# rust-dummy

This is a template demonstrating how to build a QEMU QOM device using a hybrid C/Rust approach, providing an answer to the "C/Rust peripheral interop story" without relying on full QEMU native Rust support.

## Architecture

QEMU 10.x supports Rust natively, but its build system integration can be complex and sometimes conflicts with dynamic module loading (`--enable-modules`). This template sidesteps those issues by using raw FFI:

1. **`src/lib.rs`**: The actual device behavior. It exports simple `extern "C"` functions for `read` and `write`.
2. **`rust-dummy.c`**: The QOM boilerplate. It registers the `TypeInfo`, handles the `MemoryRegionOps`, and forwards memory accesses to the Rust functions.
3. **`meson.build` integration**: The Rust code is compiled as a `staticlib` via a `custom_target` executing `rustc` directly. The resulting `.a` archive is then linked into the `hw-virtmcu-rust-dummy.so` module generated by Meson. A dummy C file is generated to satisfy QEMU's `modinfo-collect.py` which only scans C files.

## Usage

Load the device in QEMU like any other module:

```bash
qemu-system-arm ... -device rust-dummy,base-addr=0x60000000
```

Any reads to `0x60000000` will be intercepted by the Rust implementation (which currently returns `0xdeadbeef` for offset `0`).
